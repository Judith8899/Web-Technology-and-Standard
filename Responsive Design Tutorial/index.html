Responsive Design

Nowadays, we are facing various electronic devices, such as laptop, mobile phone, tablet PC, and so on. Have you ever notice that there are different kinds of display when we browse same website in different devices? It's all about a website designing technology called responsive design. Because the original intention of responsive design is to fit a variety of windows of electronic devices.

HTML5 Overview

HTML5 is from cooperation between W3C (World Wide Web Consortium) and WHATWG (Web Hypertext Application Technology Working Group).

In HTML5, there are some rules:

  • 1. Those new characteristics are supposed to be based on HTML, CSS, DOM and JavaScript.
  • 2. Reduce the independency on external plug-ins(Such as Flash).
  • 3. Excellent ability in exception processing
  • 4. HTML5 is supposed to be independent from devices
  • 5. Development progress should be transparent

In HTML5, there are a variety of interesting characteristics:

  • 1. Canvas
  • 2. Video &Audio - Jump To Media Playback
  • 3. Better support for local offline storage
  • 4. New special content elements: article, footer, header and section
  • 5. New sheet controls: calendar, date, time, email, url and search

Canvas

What is Canvas?

Canvas is used to draw pictures in website using JavaScript. Canvas is a rectangular area, and we can control every pixel in it.Many methods are encapsulated in Canvas, such drawing path, rectangular, circle, string and inserting pitcures.

How to create Canvas?

<html>
<head> 
</script>
</head>
<body>
<canvas  id="myCanvas"></canvas>
</body>
</html>

How to draw in JavaScript?

Actually, we can not draw directly through Canvas. All of drawing work are supposed to be completed in JavaScript:

<html>

<head>  
</script>
</head>
<body>

  <script type="text/javascript">
  var canvas=document.getElementById('myCanvas');
  var ctx=canvas.getContext('2d');
  ctx.fillStyle='#FF0000';
  ctx.fillRect(0,0,80,100);
  </script>

</body>
</html>

Firstly, locate <canvas>: 


 var c=document.getElementById("myCanvas");

Then, create context object:

 var ctx=c.getContext("2d");

getContext("2d") is a build-in object of HTML5, with various methods to draw lines, rectangles, circles, characters and pictures. Here's an example to draw a red rectangle:

Offline Storage Database

What is offline web application?

An application cache mechanism is provided with HTML5 and it can support Web application to run offline. Developers can specify whether browser caches are available in an offline state. In this condition, applications which is already cached can still run when users reopen or restart it.

Why use offline web application?

Offline browsing: Users still have access to website even in an offline condition.Speed: It would be faster because cached resources saved locally.Reduce server load: Only renewed content need to be loaded.

Foundation of Cache Manifest

To enable application caching, in the document, there is a manifest attribute：

 <html manifest="demo.appcache">
 .....
 </html>

Each page that specifies the manifest is cached when the user accesses it. If the manifest attribute is not specified, the page is not cached (unless the page is specified directly in the manifest file). The proposed file extension for the manifest file is: ".appcache".

Manifest File

Manifest file is a simple text file, It tells the browser what is cached (and what is not cached). Manifest file has three parts:

  • 1.CACHE MANIFEST - The following files will be cached after the first download.
  • 2.NETWORK - The following files in this title need to be connected to the server and will not be cached.
  • 3.FALLBACK - The following file specifies the return page (such as a 404 page) when the page is not accessible

CACHE MANIFEST

The first line, CACHE MANIFEST, is necessary:

CACHE MANIFEST
/theme.css
/logo.gif
/main.js

The manifest file above lists three resources:A CSS file, a GIF image, and a JavaScript file. When the manifest file is loaded, the browser downloads the three files from the root directory of the site. Then, no matter when the user is disconnected from the Internet, these resources are still available.

NETWORK

The following NETWORK section specifies that the file "login.php" will never be cached and is not available offline:

NETWORK:
login.php

FALLBACK

The following FALLBACK section specifies that if the Internet connection cannot be established, all the files in the "offline.html" alternative /html5/ directory will be used:

FALLBACK:
/html/ /offline.html

Update the cache

Once the application is cached, it maintains the cache until the following occurs.

  • 1. The user clears the browser cache.
  • 2. The manifest file is modified (see the prompt below).
  • 3. The manifest file is modified (see the prompt below)

Browser History Management

Why use browser history management?

In latest web application, AJAX partial page update is domain. Therefore, it is meaningless for “show previous page” and “show next page”. However, those history can be managed in HTML5 by refreshing those history elements.

HTML5 history API

HTML5 history API only has 2 methods: history.pushState() and history.replaceState()，and one event: window.onpopstate。

How to implement?

First, add support for the URL state parameter on the server side, for example: page=3 will output the content of the page number (back end template). It can also be the server side that gives the data to the JavaScript, which is written to the page by JavaScript (the front-end template). Next, use history. PushState (), and at the same time, the correct URL with parameters is set. The code might look like this: 

newURL = "?page=" + pageNow;
history.pushState(null, "", newURL);

Add the popstate event to the window, and add the processing when the navigation changes. The code looks like this:

$(window).on("popstate", function(event) {
    var state = event.originalEvent.state,
        reg = /page=(\d+)/,
        regMatch = reg.exec(location.search),
        pageNow = regMatch === null ? 1 : +regMatch[1];
    updateByPage(pageNow);
});

So, we're done. In the browser that supports the HTML5 history API, the above sections have already done the page numbering of Ajax pages.

Drag and Drop

What is Drag and Drop?

We can arrive in the aim that drag one element and drop in another place.

Set the element to drag and drop

Firstly, to make the element draggable, set the draggable attribute to true:

<img draggable="true">

ondragstart and setData()

In the example above, the ondragstart attribute calls a function, drag(event), which specifies the data to be dragged. The datatransfer.setdata () method sets the data type and value of the dragged data:

function drag(ev)
{
    ev.dataTransfer.setData("Text",ev.target.id);
}

ondragover

ondragover event specifies where to place the dragged data. By default, data/elements cannot be placed in other elements. If you need to set permission to place, we must block the default handling of the element. If you need to set permission to place, we must block the default handling of the element. This is done by calling the event.preventdefault () of the ondragover event:

event.preventDefault()

ondrop

Drop events occur when the data is being dragged. In the example above, the ondrop attribute calls a function, drop(event):

function drop(ev)
{
    ev.preventDefault();
    var data=ev.dataTransfer.getData("Text");
    ev.target.appendChild(document.getElementById(data));
}
